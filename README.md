🚀 Интеграция отзывов Яндекс Карт через Apify (Laravel + Vue)
Система автоматического сбора, хранения и отображения отзывов из Яндекс Карт. Решение построено на стеке Laravel 11 и Vue 3 (Inertia.js) с использованием сервиса Apify для обхода ограничений API Яндекса.

📋 Обзор системы
Проект решает задачу мониторинга репутации организации, позволяя импортировать отзывы в локальную базу данных для быстрого доступа и анализа.

Архитектура данных
Яндекс Карты ➔ Apify Scraper ➔ Laravel DB ➔ Vue.js UI

Основные возможности
✅ Автоматизация: Загрузка до 1000 отзывов за один цикл.

✅ Производительность: Хранение в БД (MySQL/PostgreSQL) — мгновенная работа интерфейса без ожиданий API.

✅ Аналитика: Вывод общего рейтинга компании и общего количества отзывов.

✅ Гибкость: Сортировка (по дате, по рейтингу) и удобная пагинация.

✅ Медиа: Поддержка отображения фотографий пользователей из отзывов.

✅ Безопасность: API токен хранится строго на бэкенде.

🛠 Требования
PHP: 8.2+

Node.js: 18+

Database: Любая поддерживаемая Laravel (рекомендуется MySQL).

Apify Account: Активный токен (Free или Starter).

🔑 Регистрация и получение API ключа (Apify)
Для работы системы необходимо получить персональный токен доступа:

Зарегистрируйтесь на сайте Apify.com.

Перейдите в раздел Settings -> Integrations (или по ссылке console.apify.com/account/integrations).

Скопируйте ваш Personal API token.

Вставьте полученный ключ в файл .env вашего проекта (переменная APIFY_API_TOKEN).

⚙️ Установка и запуск
Следуйте этим шагам для развертывания проекта с нуля:

1. Клонирование репозитория
Bash
git clone https://github.com/hezam17/yandex-maps-integration.git
cd yandex-maps-integration.git
2. Установка зависимостей
Bash
composer install
npm install
3. Настройка окружения
Скопируйте пример конфига и сгенерируйте ключ:

Bash
cp .env.example .env
php artisan key:generate
4. Конфигурация .env
Откройте .env и добавьте ваши данные базы данных, а также ключи Apify:

# База данных
DB_CONNECTION=mysql
DB_DATABASE=your_db_name

# Apify Настройки
# Используйте ~ вместо / для ID актора
APIFY_API_TOKEN=apify_api_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
APIFY_ACTOR_ID=zen-studio~yandex-maps-scraper

5. Миграция базы данных
Bash
php artisan migrate
6. Сборка фронтенда и запуск
Bash
npm run dev
# В другом терминале
php artisan serve
🔄 Синхронизация данных
Как запустить сбор данных?
Зайдите в раздел Настройки (Settings) в интерфейсе.

Вставьте прямую ссылку на вкладку "Отзывы" вашей организации в Яндекс Картах.

Пример: https://yandex.com/maps/46/kirov/?indoorLevel=1&ll=49.682104%2C58.602576&mode=poi&poi%5Bpoint%5D=49.680759%2C58.603278&poi%5Buri%5D=ymapsbm1%3A%2F%2Forg%3Foid%3D81817248367&tab=reviews&z=18

Нажмите Сохранить. Первая синхронизация запустится автоматически.

Планировщик (Cron)
Для автоматического обновления отзывов (раз в сутки в 03:00) добавьте запись в CRON вашего сервера:

Bash
* * * * * cd /var/www/your-project && php artisan schedule:run >> /dev/null 2>&1
Ручные команды (Artisan)
Полная синхронизация всех компаний:

Bash
php artisan yandex:sync-reviews
Принудительное обновление (игнорируя кэш):

Bash
php artisan yandex:sync-reviews --force
🖼 Интерфейс пользователя
Рейтинговые бейджи: Цветовая индикация (Зеленый 4-5, Желтый 3, Красный <3).

Пагинация: По 10 отзывов на страницу с быстрой навигацией.

Обновление: Кнопка ручного обновления в реальном времени.

🚀 Важные советы для разработчика (Inspiration)
Экономия лимитов: Система считает данные актуальными в течение 6 часов. Это защищает ваш баланс в Apify от лишних трат при случайных нажатиях кнопки "Обновить".

Обработка ошибок: Если Apify упадет или лимит будет исчерпан, система покажет последнее сохраненное состояние из БД, не прерывая работу сайта.

Логирование: Все важные события синхронизации записываются в storage/logs/yandex-sync.log. Проверяйте его, если отзывы перестали поступать.

Масштабируемость: Вы можете добавить несколько организаций, и система будет обрабатывать их по очереди через консольную команду.

🛡 Безопасность
Rate Limiting: Ручная синхронизация ограничена 3 запросами в час для одного пользователя.

Data Integrity: Использование upsert (обновление или вставка) по уникальному external_id исключает дублирование отзывов.
